{
    "id":"1", //gui id 不可重复
    "layer":"bg", //当前控件显示在前景层还是背景层，前景层永远在背景层之前
    "className":"UIImageView/UIButton/UILabel", //当前控件的类名
    "relativeLayout":"1,1,1,1,0,0",//布局样式，1则为按屏幕比例的相对布局 0为按point的绝对布局，6个值分别对应 anchorPoint 2个及frame 4个
    "anchorPoint":"0.5,1",//控件的坐标原点，以左上角为（0，0），无论相对还是绝对，都是以这个点为基准
    "frame":"0.5,1,200,150",//控件的frame，相对布局与绝对布局可混用
    "frameOffsetInPoint":"0,0,0,0",//控件frame的偏移量，只能是以point为单位的值，这四个值会分别叠加在最终计算出的frame的四个值里
    "lockAspect":"0",//是否锁定宽高比，若锁定，则所有frame的height的设置失效，以width按比例来计算
    "hwAspectRatio":"1",//height/width的值，锁定宽高比时使用
    "zOrder":"90",//z轴的值，大的在前面，挡住后面的，前景层与背景层的互不影响
    "backgroundColor":"255,255,255,255",//背景颜色值，rgba，0~255,
    "hide":"1",
}

关于forwardLogic和backwardLogic:

    当指令之间的执行存在冲突时，彼此间的逻辑由 队列前面的指令的 backwardLogic和队列后面的指令的forwardLogic决定( 表格内容格式为 前指令行为/后指令行为)


waitperformNowforceCancelAllforceCancelSamecancelSamecancelSelf
beCacnelled继续执行/等待继续执行/同时执行被取消/执行若为同类型则被取消/执行若为同类型则被取消/执行继续执行/被取消
notBeCancelled继续执行/等待继续执行/同时执行被取消/执行若为同类型则被取消/执行继续执行/等待继续执行/被取消
cancelBackward继续执行/被取消继续执行/被取消被取消/执行若为同类型则被取消/执行继续执行/被取消继续执行/被取消
